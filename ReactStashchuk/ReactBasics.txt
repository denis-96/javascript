Основы React

Document Object Model (DOM)
Имеет древовидную структуру
Его обновление - затратная операция

Виртуальный DOM - объект JavaScript
Обновление виртуального DOM происходит практически мгновенно
При изменении состояния приложения меняется сначала 
виртуальный дом, а потом реальный

После изменения состояния элементов React поэлементно сравнивает
текущее состояние вирт ДОМ с предыдущем состоянием вирт ДОМ
Он определяет отличия и вносит изменения в реальный ДОМ
(меняются только опеределённые элементы, а не перерисовывается вся страница)

Ключевые понятия в React:
Компоненты (components)
JSX (JavaScript Syntax Extension)
Свойства (props)
Состояния (state)


Компоненты

Характеристики компонентов:
1. Компоненты можно использовать многократно
2. Компоненты формируют четкую иерархическую структуру
3. Видом компонентов можно управлять извне с помощью свойств
4. Компоненты могут иметь своё собственное состояние
5. Каждый компонент отвечает за отдельную логическую часть интерфейса

Анатомия компонента React:
(HTML, CSS, JavaScript) - компонент React


Функциональные Компоненты:

function HelloWorld() {
    return <h1>Hello World</h1>
}

или

const HelloWorld = () => {
    return <h1>Hello World</h1>
}

Классовые компоненты

class HelloWorld extends Component {
    render() {
        return <h1>Hello World</h1>
    }
}

На данный момент чаще всего используются функциональные компоненты


JSX - JavaScript Syntax Extension

Специальный синтаксис, с помощью которого можно объединять
JavaScript, HTML и CSS,
тем самым формировать полноценные компоненты React
Синтаксис JSX похож на синтаксис HTML и CSS

JSX - синтаксическая надстройка над JS


Пользовательские и встроенные компоненты в JSX
Встроенные компоненты: div, h1, p и тд

Пользовательские компоненты - компоненты, которые создаёт разработчик
Названия пользовательских компонентов должны начинаться с большой буквы

JSX должен иметь один корневой элементов


Свойства и состояния (props and state)

Снаружи компонент react получает свойства
Внутри компонента react может быть состояние
Состояние может изменяться на протяжении жизненного цикла компонента

Каждый компонент может передават свойства дочерним компонентам, 
если таковые имеются
У одного компонента может быть несколько дочерних
